# M7 - Analityka i pomiar

## Cel

Twoja strona-system jest już w internecie (M6). Teraz dodajesz pomiar: chcesz wiedzieć, ile osób wchodzi, skąd przychodzą i czy Google widzi Twoją stronę. Skill `analityka` wpina to za Ciebie - Ty podajesz tylko ID, które Ci pokaże, gdzie znaleźć.

## Po module

Masz:

- licznik ruchu (Vercel Analytics) - widzisz, ile osób było na stronie,
- stronę zgłoszoną w Google Search Console + sitemap i robots,
- self-canonical (Google nie liczy Twoich podstron jako duplikatów),
- pełną analitykę GA4 przez Google Tag Manager,
- opcjonalnie Meta Pixel, jeśli planujesz reklamy,
- prosty baner zgody na cookies i notkę o prywatności.

## Zanim zaczniesz

Ten moduł ma sens dopiero, gdy strona jest opublikowana (M6). Tracking widać na żywo dopiero po deployu.

## Uruchom skill

```plain text
Uruchom skill analityka.

Moja strona jest juz opublikowana. Dodaj prosty pomiar: licznik ruchu, Google Analytics, Search Console i mape strony. Prowadz mnie krok po kroku, mow dokladnie, skad wziac potrzebne ID. Nie pytaj mnie o decyzje techniczne.
```

## Jak to działa

Skill idzie od najłatwiejszego do pełnego:

1. **Vercel Analytics** - licznik ruchu od ręki, bez żadnych ID.
2. **Search Console + sitemap + robots** - żeby Google szybciej znalazł Twoje podstrony.
3. **Self-canonical** - żeby Google nie widział duplikatów.
4. **GA4 przez Google Tag Manager** - pełna analityka: skąd przychodzą ludzie, co klikają.
5. **Meta Pixel** - tylko jeśli robisz reklamy na Facebooku/Instagramie.
6. **Baner zgody na cookies + notka prywatności** - bo dodajesz narzędzia analityczne.

Po każdym kroku skill robi commit, więc nic nie przepada.

## Gdy nie wiesz, skąd wziąć ID

Skill powie Ci dokładnie, gdzie kliknąć. Jeśli i tak utkniesz, otwórz `prompty/research-perplexity.md` (sekcja "Analityka - skad wziac ID"), wklej gotowy prompt do Perplexity i wróć z numerem.

## Ważne po deployu

ID trackingu (na przykład `NEXT_PUBLIC_GTM_ID`) muszą trafić RĘCZNIE do Vercel w ustawieniach zmiennych środowiskowych, a potem trzeba zrobić deploy. Inaczej analityka nie zadziała na żywej stronie. Po deployu sprawdź w GA4 zakładkę Realtime - powinieneś/powinnaś zobaczyć swoją własną wizytę.
