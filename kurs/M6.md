# M6 - Publikacja: Twoja strona w internecie

## Cel

Publikujesz strone w internecie przez GitHub i Vercel.

## Brama przed publikacja

```plain text
Uruchom skill sprawdz-kod. Chce wypchnac strone na GitHub i Vercel, wiec sprawdz, czy build przejdzie, przelec checklistę anti-ai-look i zrob brame bezpieczenstwa: sekrety, .env.local, NEXT_PUBLIC, formularz/API i zaleznosci.

Na koncu daj mi krotka liste: OK do publikacji / rzeczy do poprawy przed publikacja.
```

## GitHub

GitHub trzyma kod strony i historie zmian.

```plain text
Przygotuj projekt do wrzucenia na GitHub. Sprawdz status gita, pokaz pliki, ktore trafia do commita, upewnij sie, ze .env.local nie trafi do repozytorium, a potem przygotuj pierwszy commit z jasnym opisem.
```

## Vercel

Vercel publikuje strone z GitHuba.

W panelu Vercel:

1. `Add New Project`.
2. Wybierz repozytorium.
3. Zostaw Next.js, jesli Vercel wykryje automatycznie.
4. Dodaj zmienne srodowiskowe.
5. Kliknij deploy.

## Test po deployu

Sprawdz:

- czy strona sie laduje,
- czy dziala mobile,
- czy formularz wysyla mail,
- czy obrazy sie laduja,
- czy linki dzialaja,
- czy nie ma placeholderow,
- czy nie ma sekretow w publicznym kodzie.

Prompt:

```plain text
Mam opublikowana strone pod tym adresem: [WKLEJ LINK].
Sprawdz ja jak strone produkcyjna: desktop, mobile, formularz, bledy w konsoli, obrazy, anti-AI-look i podstawowe ryzyka bezpieczenstwa. Wypisz tylko rzeczy, ktore realnie trzeba poprawic przed pokazaniem klientom.
```

## Domena

Gdy link z Vercel dziala, podepnij domene w panelu Vercel. Po podpieciu domeny sprawdz strone jeszcze raz, szczegolnie formularz i `https://`.

## Efekt koncowy M6

Masz publiczny link do strony i wiesz, jak publikowac kolejne zmiany.
