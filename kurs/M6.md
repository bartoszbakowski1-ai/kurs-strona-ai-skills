# M6 - Publikacja: Twoja strona w internecie

## Cel

Publikujesz stronę w internecie przez GitHub i Vercel.

## Brama przed publikacją

```plain text
Uruchom skill sprawdz-kod. Chce wypchnac strone na GitHub i Vercel, wiec sprawdz, czy build przejdzie, przelec checklistę anti-ai-look i zrob brame bezpieczenstwa: sekrety, .env.local, NEXT_PUBLIC, formularz/API i zaleznosci.

Na koncu daj mi krotka liste: OK do publikacji / rzeczy do poprawy przed publikacja.
```

## GitHub

GitHub trzyma kod strony i historię zmian.

```plain text
Przygotuj projekt do wrzucenia na GitHub. Sprawdz status gita, pokaz pliki, ktore trafia do commita, upewnij sie, ze .env.local nie trafi do repozytorium, a potem przygotuj pierwszy commit z jasnym opisem.
```

## Vercel

Vercel publikuje stronę z GitHuba.

W panelu Vercel:

1. `Add New Project`.
2. Wybierz repozytorium.
3. Zostaw Next.js, jeśli Vercel wykryje automatycznie.
4. Dodaj zmienne środowiskowe.
5. Kliknij deploy.

## Test po deployu

Sprawdź:

- czy strona się ładuje,
- czy działa mobile,
- czy formularz wysyła mail,
- czy obrazy się ładują,
- czy linki działają,
- czy nie ma placeholderów,
- czy nie ma sekretów w publicznym kodzie.

Prompt:

```plain text
Mam opublikowana strone pod tym adresem: [WKLEJ LINK].
Sprawdz ja jak strone produkcyjna: desktop, mobile, formularz, bledy w konsoli, obrazy, anti-AI-look i podstawowe ryzyka bezpieczenstwa. Wypisz tylko rzeczy, ktore realnie trzeba poprawic przed pokazaniem klientom.
```

## Domena - podepnij własny adres (2 rekordy DNS)

Strona działa już pod adresem `nazwa.vercel.app`. Żeby miała Twój własny adres (np. `twojemie.pl`), łączysz domenę z Vercel. W praktyce sprowadza się to do zalogowania się u dostawcy domeny (tam, gdzie ją kupiłaś/kupiłeś) i ustawienia DWÓCH rekordów DNS.

### Krok 1 - dodaj domenę w Vercel
W projekcie na Vercel: `Settings` -> `Domains` -> `Add` -> wpisz swoją domenę (i wersję z `www`). Vercel od razu pokaże, jakie rekordy DNS masz ustawić.

### Krok 2 - przepisz wartości, które pokaże Vercel
Zwykle są to dokładnie dwa rekordy:
- **rekord A** dla gołej domeny (nazwa: `@`), wartość zwykle `76.76.21.21`,
- **rekord CNAME** dla `www` (nazwa: `www`), wartość zwykle `cname.vercel-dns.com`.

WAŻNE: bierz dokładnie te wartości, które pokazuje Vercel u Ciebie - czasem są inne (Vercel daje czasem indywidualny adres). Nie przepisuj z pamięci, przepisuj z ekranu Vercel.

### Krok 3 - ustaw te 2 rekordy u dostawcy domeny
Zaloguj się tam, gdzie kupiłaś/kupiłeś domenę, znajdź "Strefę DNS" tej domeny i dodaj albo popraw te dwa rekordy. Gdzie to jest u najpopularniejszych dostawców:

- **home.pl**: Panel klienta -> `Domeny` (menu z lewej) -> kliknij nazwę domeny -> moduł `Hosting DNS` -> `Opcje` -> `Zarządzaj rekordami DNS` -> `Dodaj nowy rekord`.
- **nazwa.pl**: `nazwa.pl/panel` -> `Uslugi` -> `Domeny` -> przy domenie `konfiguruj` -> zakładka `Reczna konfiguracja DNS` -> przycisk `ZMIEN` -> `Dodaj kolejny rekord`.
- **OVHcloud**: `manager.ovh.com` -> `Web Cloud` -> `Domeny` -> wybierz domenę -> zakładka `Strefa DNS` -> `Dodaj wpis`.
- **cyberFolks**: Panel klienta -> `Domeny` -> wybierz domenę -> `Strefa DNS` (edytuj rekordy).
- **Hostinger**: hPanel -> `Domeny` -> wybierz domenę -> `Strefa DNS` (Rekordy DNS).
- **zenbox.pl**: Panel -> `Domeny` -> wybierz domenę -> `Strefa DNS`.
- **dhosting.pl**: Panel -> `Domeny` -> wybierz domenę -> `Edytor DNS` / `Strefa DNS`.
- **AZ.pl**: Panel -> `Domeny` -> wybierz domenę -> `Strefa DNS`.
- **LH.pl**: Panel -> `Domeny` -> wybierz domenę -> `Strefa DNS`.
- **GoDaddy** (zagraniczny): konto -> `Moje produkty` -> przy domenie `DNS` -> `Zarządzaj strefami DNS` -> sekcja `Rekordy` -> dodaj/edytuj.

Nazwy w panelach czasem się zmieniają. Jeśli nie widzisz dokładnie takiej nazwy, szukaj w panelu słowa `DNS` albo `Strefa DNS`. Twojego dostawcy nie ma na liście? Użyj promptu z `prompty/research-perplexity.md` albo zapytaj wprost:

```plain text
Mam domene u dostawcy [NAZWA DOSTAWCY]. Wytlumacz mi krok po kroku, prostym jezykiem, gdzie w panelu znajde strefe DNS tej domeny i jak dodac rekord A oraz rekord CNAME. Nie jestem osoba techniczna.
```

### Krok 4 - poczekaj i sprawdź
Po zapisaniu rekordów zmiany potrzebują chwili (zwykle kilka minut, czasem do kilku godzin). Vercel sam wykryje poprawną konfigurację, przy domenie pokaże `Valid Configuration` i automatycznie wystawi certyfikat HTTPS (kłódka w przeglądarce). Gdy domena działa, sprawdź stronę jeszcze raz, szczególnie formularz i to, czy adres ma `https://`.

Uwaga: niektórzy dostawcy proponują zamiast tego zmianę serwerów nazw (nameservers) na serwery Vercel. To też działa, ale prościej jest zostawić serwery dostawcy i dodać tylko te dwa rekordy.

## Efekt końcowy M6

Masz publiczny link do strony i wiesz, jak publikować kolejne zmiany.
