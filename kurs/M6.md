# M6 - Publikacja: Twoja strona w internecie

## Cel

Publikujesz strone w internecie przez GitHub i Vercel.

## Brama przed publikacja

```plain text
Uruchom skill sprawdz-kod. Chce wypchnac strone na GitHub i Vercel, wiec sprawdz, czy build przejdzie, przelec checklistę anti-ai-look i zrob brame bezpieczenstwa: sekrety, .env.local, NEXT_PUBLIC, formularz/API i zaleznosci.

Na koncu daj mi krotka liste: OK do publikacji / rzeczy do poprawy przed publikacja.
```

## GitHub

GitHub trzyma kod strony i historie zmian.

```plain text
Przygotuj projekt do wrzucenia na GitHub. Sprawdz status gita, pokaz pliki, ktore trafia do commita, upewnij sie, ze .env.local nie trafi do repozytorium, a potem przygotuj pierwszy commit z jasnym opisem.
```

## Vercel

Vercel publikuje strone z GitHuba.

W panelu Vercel:

1. `Add New Project`.
2. Wybierz repozytorium.
3. Zostaw Next.js, jesli Vercel wykryje automatycznie.
4. Dodaj zmienne srodowiskowe.
5. Kliknij deploy.

## Test po deployu

Sprawdz:

- czy strona sie laduje,
- czy dziala mobile,
- czy formularz wysyla mail,
- czy obrazy sie laduja,
- czy linki dzialaja,
- czy nie ma placeholderow,
- czy nie ma sekretow w publicznym kodzie.

Prompt:

```plain text
Mam opublikowana strone pod tym adresem: [WKLEJ LINK].
Sprawdz ja jak strone produkcyjna: desktop, mobile, formularz, bledy w konsoli, obrazy, anti-AI-look i podstawowe ryzyka bezpieczenstwa. Wypisz tylko rzeczy, ktore realnie trzeba poprawic przed pokazaniem klientom.
```

## Domena - podepnij wlasny adres (2 rekordy DNS)

Strona dziala juz pod adresem `nazwa.vercel.app`. Zeby miala Twoj wlasny adres (np. `twojemie.pl`), laczysz domene z Vercel. W praktyce sprowadza sie to do zalogowania sie u dostawcy domeny (tam, gdzie ja kupilas/kupiles) i ustawienia DWOCH rekordow DNS.

### Krok 1 - dodaj domene w Vercel
W projekcie na Vercel: `Settings` -> `Domains` -> `Add` -> wpisz swoja domene (i wersje z `www`). Vercel od razu pokaze, jakie rekordy DNS masz ustawic.

### Krok 2 - przepisz wartosci, ktore pokaze Vercel
Zwykle sa to dokladnie dwa rekordy:
- **rekord A** dla golej domeny (nazwa: `@`), wartosc zwykle `76.76.21.21`,
- **rekord CNAME** dla `www` (nazwa: `www`), wartosc zwykle `cname.vercel-dns.com`.

WAZNE: bierz dokladnie te wartosci, ktore pokazuje Vercel u Ciebie - czasem sa inne (Vercel daje czasem indywidualny adres). Nie przepisuj na pamiec, przepisuj z ekranu Vercel.

### Krok 3 - ustaw te 2 rekordy u dostawcy domeny
Zaloguj sie tam, gdzie kupilas/kupiles domene, znajdz "Strefe DNS" tej domeny i dodaj albo popraw te dwa rekordy. Gdzie to jest u najpopularniejszych dostawcow:

- **home.pl**: Panel klienta -> `Domeny` (menu z lewej) -> kliknij nazwe domeny -> modul `Hosting DNS` -> `Opcje` -> `Zarzadzaj rekordami DNS` -> `Dodaj nowy rekord`.
- **nazwa.pl**: `nazwa.pl/panel` -> `Uslugi` -> `Domeny` -> przy domenie `konfiguruj` -> zakladka `Reczna konfiguracja DNS` -> przycisk `ZMIEN` -> `Dodaj kolejny rekord`.
- **OVHcloud**: `manager.ovh.com` -> `Web Cloud` -> `Domeny` -> wybierz domene -> zakladka `Strefa DNS` -> `Dodaj wpis`.
- **cyberFolks**: Panel klienta -> `Domeny` -> wybierz domene -> `Strefa DNS` (edytuj rekordy).
- **Hostinger**: hPanel -> `Domeny` -> wybierz domene -> `Strefa DNS` (Rekordy DNS).
- **zenbox.pl**: Panel -> `Domeny` -> wybierz domene -> `Strefa DNS`.
- **dhosting.pl**: Panel -> `Domeny` -> wybierz domene -> `Edytor DNS` / `Strefa DNS`.
- **AZ.pl**: Panel -> `Domeny` -> wybierz domene -> `Strefa DNS`.
- **LH.pl**: Panel -> `Domeny` -> wybierz domene -> `Strefa DNS`.
- **GoDaddy** (zagraniczny): konto -> `Moje produkty` -> przy domenie `DNS` -> `Zarzadzaj strefami DNS` -> sekcja `Rekordy` -> dodaj/edytuj.

Nazwy w panelach czasem sie zmieniaja. Jesli nie widzisz dokladnie takiej nazwy, szukaj w panelu slowa `DNS` albo `Strefa DNS`. Twojego dostawcy nie ma na liscie? Uzyj promptu z `prompty/research-perplexity.md` albo zapytaj wprost:

```plain text
Mam domene u dostawcy [NAZWA DOSTAWCY]. Wytlumacz mi krok po kroku, prostym jezykiem, gdzie w panelu znajde strefe DNS tej domeny i jak dodac rekord A oraz rekord CNAME. Nie jestem osoba techniczna.
```

### Krok 4 - poczekaj i sprawdz
Po zapisaniu rekordow zmiany potrzebuja chwili (zwykle kilka minut, czasem do kilku godzin). Vercel sam wykryje poprawna konfiguracje, przy domenie pokaze `Valid Configuration` i automatycznie wystawi certyfikat HTTPS (klodka w przegladarce). Gdy domena dziala, sprawdz strone jeszcze raz, szczegolnie formularz i to, czy adres ma `https://`.

Uwaga: niektorzy dostawcy proponuja zamiast tego zmiane serwerow nazw (nameservers) na serwery Vercel. To tez dziala, ale prosciej jest zostawic serwery dostawcy i dodac tylko te dwa rekordy.
