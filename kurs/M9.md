# M9 - Kontrola i rozwój: zmiany w 15 minut

## Cel

Umiesz utrzymywać stronę po publikacji: robić małe zmiany, dodawać podstrony, wznawiać pracę i bezpiecznie cofać nieudane zmiany.

## Status

```plain text
/strona status
```

Jeśli komenda nie działa:

```plain text
Sprawdz status mojego projektu strony: aktualny cel, najwazniejsze pliki, ostatnie zmiany w git, czy sa niecommitowane zmiany i co powinienem/powinnam zrobic jako kolejny bezpieczny krok.
```

## Wznowienie pracy

```plain text
/strona wznow
```

Albo:

```plain text
Wznow prace nad ta strona. Najpierw przeczytaj README, PROGRESS.md, karty strategiczne i ostatni status gita. Potem powiedz mi, na jakim etapie jestesmy i jaki jest jeden najlepszy kolejny krok.
```

## Nowa podstrona

```plain text
/strona nowa-podstrona
```

Jeśli nie używasz komendy slash:

```plain text
Dodaj nowa podstrone: [NAZWA].
Jej cel to: [CEL].
Ma pasowac do obecnego stylu strony, uzywac istniejacych komponentow i nie dublowac tresci ze strony glownej.
Najpierw zaproponuj strukture, potem po mojej akceptacji wdrozenie.
```

## Mała zmiana tekstu

```plain text
Zmien tylko copy w sekcji [NAZWA SEKCJI].
Obecnie brzmi: [WKLEJ].
Nowa wersja ma komunikowac: [INTENCJA].
Nie zmieniaj layoutu ani komponentow.
```

## Przed publikacją zmiany

```plain text
Przed publikacja tej zmiany uruchom sprawdz-kod. Sprawdz build, mobile, formularz i czy zmiana nie zepsula istniejacych sekcji. Jesli jest OK, przygotuj commit z krotkim opisem.
```

## Cofanie nieudanej zmiany

```plain text
Ostatnia zmiana zepsula strone. Znajdz ostatni dzialajacy punkt kontrolny i zaproponuj bezpieczne cofniecie przez git revert, bez kasowania historii i bez git reset --hard.
```

## Plan na 7 dni

- pokaż stronę 3 osobom z grupy docelowej,
- popraw jedno niejasne miejsce w ofercie,
- dodaj jeden dowód: opinie, screen, case albo wynik,
- sprawdź formularz,
- popraw hero albo CTA,
- dodaj jedną odpowiedź na realną obiekcję,
- opublikuj finalną wersję i zacznij ją wysyłać.

## Efekt końcowy M9

Masz nie tylko stronę, ale system pracy ze stroną.
