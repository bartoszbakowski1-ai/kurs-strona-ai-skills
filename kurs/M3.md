# M3 - 3 karty: dla kogo i jak ma wygladac

## Cel

Tworzysz trzy karty strony, zanim Claude Code zacznie pisac kod.

Efekt:

```plain text
karty/
  karta-strategiczna.md
  karta-architektury-tresci.md
  karta-wizualna.md
```

## Po co sa trzy karty

- **Karta strategiczna** - dla kogo jest strona, co obiecuje, jaka jest glowna akcja.
- **Karta architektury tresci** - jakie sekcje i podstrony powinny powstac.
- **Karta wizualna** - jak strona ma wygladac i czego unikac.

## Uruchom skill

```plain text
Uruchom skill karty.

Przeczytaj najpierw folder kontekst/. Potem przeprowadz mnie przez stworzenie trzech kart strony. Pytaj po ludzku, jedno zagadnienie naraz. Na koncu zapisz pliki w folderze karty/.
```

## Wazne pytania

Przygotuj odpowiedzi:

- kto ma wejsc na strone,
- co ma zrozumiec w pierwszych 10 sekundach,
- jaka jest jedna glowna akcja,
- jakie masz dowody wiarygodnosci,
- jakie sekcje musza byc na stronie,
- jakiego stylu wizualnego chcesz unikac,
- jakie masz zdjecia, screeny, mockupy albo inne media.

## Bramka mediow

Minimum: 3 konkretne media.

Lepiej: 5-7 mediow dla dluzszej strony.

Media to nie tylko zdjecia twarzy. To moga byc:

- zdjecia Ciebie,
- zdjecia procesu pracy,
- screeny produktu,
- mockupy,
- realizacje,
- fragmenty materialow,
- zdjecia z warsztatow,
- opinie jako screenshoty.

## Sprawdz karty

Po zakonczeniu:

```plain text
Przeczytaj trzy karty i ocen, czy wystarcza do budowy strony. Wskaz sprzecznosci, zbyt ogolne miejsca, brak dowodow i braki w mediach. Nie buduj jeszcze strony.
```

## Efekt koncowy M3

Masz brief strony. Od tego momentu Claude Code nie zgaduje, tylko buduje z dokumentow.
