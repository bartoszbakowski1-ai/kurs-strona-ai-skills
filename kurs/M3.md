# M3 - 3 karty: dla kogo i jak ma wygladac

## Cel

Projektujesz cala swoja strone-system na papierze (trzy karty), zanim Claude Code zacznie pisac kod. Projektujesz system z wielu podstron, nie pojedynczy landing.

Efekt:

```plain text
karty/
  karta-strategiczna.md
  karta-architektury-tresci.md
  karta-wizualna.md
```

## Po co sa trzy karty

- **Karta strategiczna** - po co istnieje system, jaki lejek prowadzi przez podstrony, jaka jest glowna akcja i akcje wspierajace.
- **Karta architektury tresci** - mapa systemu: jakie podstrony powstaja, jaka pelnia role, jaka jest nawigacja, jakie sekcje ma kazda podstrona i co jest trescia dynamiczna w CMS.
- **Karta wizualna** - jak strona ma wygladac, design system spojny przez wszystkie podstrony i czego unikac.

## Uruchom skill

```plain text
Uruchom skill karty.

Przeczytaj najpierw folder kontekst/. Potem przeprowadz mnie przez stworzenie trzech kart calej strony-systemu. Pytaj po ludzku, jedno zagadnienie naraz. Na koncu zapisz pliki w folderze karty/.
```

## Wazne pytania

Przygotuj odpowiedzi:

- kto ma wejsc na strone,
- co ma zrozumiec w pierwszych 10 sekundach,
- jaka jest glowna akcja i jak prowadzi do niej lejek,
- jakie podstrony tworza system (oferta, o mnie, blog/wiedza, realizacje, kontakt),
- ktora tresc chcesz pozniej edytowac sam(a) w przegladarce (blog, opinie, realizacje),
- jakie masz dowody wiarygodnosci,
- jakie sekcje musza byc na stronie glownej i podstronach,
- jakiego stylu wizualnego chcesz unikac,
- jakie masz zdjecia, screeny, mockupy albo inne media.

## Bramka mediow

Minimum: 3 konkretne media.

Lepiej: 5-7 mediow dla dluzszej strony.

Media to nie tylko zdjecia twarzy. To moga byc:

- zdjecia Ciebie,
- zdjecia procesu pracy,
- screeny produktu,
- mockupy,
- realizacje,
- fragmenty materialow,
- zdjecia z warsztatow,
- opinie jako screenshoty.

## Sprawdz karty

Po zakonczeniu:

```plain text
Przeczytaj trzy karty i ocen, czy wystarcza do budowy calego systemu. Sprawdz, czy mapa podstron, lejek i design system sa spojne. Wskaz sprzecznosci, zbyt ogolne miejsca, brak dowodow i braki w mediach. Nie buduj jeszcze strony.
```

## Efekt koncowy M3

Masz brief calej strony-systemu: mape podstron, lejek i design system. Od tego momentu Claude Code nie zgaduje, tylko buduje z dokumentow, etapami.
