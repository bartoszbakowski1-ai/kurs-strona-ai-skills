# M5 - Budowa strony-systemu + formularz

## Cel

Claude Code buduje całą Twoją stronę-system na podstawie kontekstu i trzech kart: stronę główną i podstrony z mapy, etapami. Najpierw rdzeń (strona główna + kontakt + 1-2 kluczowe podstrony), kolejne podstrony dokładasz w następnych sesjach. W obrębie każdej podstrony budujecie sekcja po sekcji.

## Zanim zaczniesz

Sprawdź, czy masz:

- `kontekst/`,
- `karty/`,
- ustawiony design z M4,
- minimum 3 media albo plan ich dodania.

Prompt kontrolny:

```plain text
Sprawdz, czy projekt jest gotowy do budowy systemu: kontekst, karty, design, media i status git. Jesli czegos brakuje, zatrzymaj sie i powiedz mi co uzupelnic.
```

## Uruchom budowę

```plain text
Uruchom skill zbuduj-strone.

Buduj moja strone-system na podstawie folderow kontekst/ i karty/.
Zacznij od rdzenia z mapy (strona glowna + kontakt + kluczowe podstrony), reszte podstron dokladaj w kolejnych etapach.
Pracuj w malych etapach: jedna sekcja albo jedna podstrona = jeden batch.
Po kazdym wiekszym etapie aktualizuj PROGRESS.md.
Nie pytaj mnie o decyzje techniczne. Pytaj tylko o tresci, oferte, dowody i media.
```

## Pierwszy ekran (hero) - tu się zatrzymaj

Hero to najważniejsza sekcja - po niej Ty i Twoi klienci ocenicie, czy strona wygląda dobrze. Claude zbuduje go jako pierwszy, z briefu hero w Karcie Wizualnej, na cały pierwszy ekran, i POKAŻE Ci go do oceny, zanim ruszy z resztą. Obejrzyj go na podglądzie i powiedz wprost, czy tak ma wyglądać. Jak się wahasz, poproś o 2-3 warianty:

```plain text
Pokaz mi hero na podgladzie. Jak sie waham, zrob 2-3 warianty pierwszego ekranu (rozny uklad i wizual, ten sam przekaz), zebym wybral(a). Reszte strony budujemy dopiero, gdy hero jest dobry.
```

## Animacje i efekt WOW

Skill budujący nakłada ruch automatycznie: każda sekcja delikatnie wjeżdża przy przewijaniu, listy korzyści kaskadą, CTA reaguje na najazd. To warstwa, która odróżnia stronę premium od płaskiego szablonu - dzieje się sama, na fundamencie z M4.

Do tego dokładacie JEDEN mocniejszy akcent na całą stronę (najlepiej na hero). Jeśli widziałeś gdzieś efekt, który Ci się podoba, podaj link:

```plain text
/skopiuj-animacje <wklej link do strony z efektem>
```

Claude otworzy tę stronę, rozpozna efekt i odtworzy go na Twojej stronie najbliżej oryginału, ale przepuści przez reguły subtelności (żeby nie spowolnił strony i nie wyglądał tanio). Zasada: jeden mocny moment na stronę, reszta to spokojny ruch. Więcej = cyrk i wolniejsza strona.

## Formularz

Gdy strona ma mieć kontakt:

```plain text
Dodaj prosty formularz kontaktowy z walidacja, bezpiecznym endpointem i konfiguracja przez zmienne srodowiskowe. Nie wpisuj sekretow w kodzie. Przygotuj .env.local.example bez prawdziwych wartosci.
```

Typowe zmienne:

```plain text
RESEND_API_KEY=...
KONTAKT_TO=...
KONTAKT_FROM=...
```

## Kontrola jakości

Po zbudowaniu strony:

```plain text
Uruchom skill sprawdz-kod. Sprawdz build, mobile, formularz, anti-ai-look, podstawy SEO (tytul i opis strony, nie placeholder), dostepnosc podstawowa i czy w kodzie nie ma sekretow.
```

## Efekt końcowy M5

Masz lokalnie działający rdzeń swojej strony-systemu (strona główna + kontakt + kluczowe podstrony) z działającym formularzem, warstwą ruchu i jednym akcentem WOW. Kolejne podstrony z mapy dokładasz tym samym cyklem. Przed publikacją przechodzisz przez bramę jakości.
