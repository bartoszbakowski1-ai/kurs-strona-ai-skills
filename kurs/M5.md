# M5 - Budowa strony-systemu + formularz

## Cel

Claude Code buduje cala Twoja strone-system na podstawie kontekstu i trzech kart: strone glowna i podstrony z mapy, etapami. Najpierw rdzen (strona glowna + kontakt + 1-2 kluczowe podstrony), kolejne podstrony dokladasz w nastepnych sesjach. W obrebie kazdej podstrony budujecie sekcja po sekcji.

## Zanim zaczniesz

Sprawdz, czy masz:

- `kontekst/`,
- `karty/`,
- ustawiony design z M4,
- minimum 3 media albo plan ich dodania.

Prompt kontrolny:

```plain text
Sprawdz, czy projekt jest gotowy do budowy systemu: kontekst, karty, design, media i status git. Jesli czegos brakuje, zatrzymaj sie i powiedz mi co uzupelnic.
```

## Uruchom budowe

```plain text
Uruchom skill zbuduj-strone.

Buduj moja strone-system na podstawie folderow kontekst/ i karty/.
Zacznij od rdzenia z mapy (strona glowna + kontakt + kluczowe podstrony), reszte podstron dokladaj w kolejnych etapach.
Pracuj w malych etapach: jedna sekcja albo jedna podstrona = jeden batch.
Po kazdym wiekszym etapie aktualizuj PROGRESS.md.
Nie pytaj mnie o decyzje techniczne. Pytaj tylko o tresci, oferte, dowody i media.
```

## Formularz

Gdy strona ma miec kontakt:

```plain text
Dodaj prosty formularz kontaktowy z walidacja, bezpiecznym endpointem i konfiguracja przez zmienne srodowiskowe. Nie wpisuj sekretow w kodzie. Przygotuj .env.local.example bez prawdziwych wartosci.
```

Typowe zmienne:

```plain text
RESEND_API_KEY=...
KONTAKT_TO=...
KONTAKT_FROM=...
```

## Kontrola jakosci

Po zbudowaniu strony:

```plain text
Uruchom skill sprawdz-kod. Sprawdz build, mobile, formularz, anti-ai-look, dostepnosc podstawowa i czy w kodzie nie ma sekretow.
```

## Efekt koncowy M5

Masz lokalnie dzialajacy rdzen swojej strony-systemu (strona glowna + kontakt + kluczowe podstrony) z dzialajacym formularzem. Kolejne podstrony z mapy dokladasz tym samym cyklem. Przed publikacja przechodzisz przez brame jakosci.
