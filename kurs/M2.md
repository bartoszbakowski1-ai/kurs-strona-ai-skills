# M2 - Framework: jak działają Twoje skille

## Cel

Rozumiesz, co masz w paczce kursowej i który skill do czego służy.

## Mapa skilli

| Skill | Kiedy użyć | Efekt |
|---|---|---|
| `kontekst` | M1 | folder `kontekst/` |
| `karty` | M3 | folder `karty/` z trzema kartami |
| `design` | M4 | system wyglądu i anti-ai-look |
| `obrazy` | M4 | realne zdjęcia i assety |
| `zbuduj-strone` | M5 | strona-system: strona główna + podstrony + formularz |
| `sprawdz-kod` | M5/M6 i po zmianach | build, mobile, jakość, bezpieczeństwo |
| `bezpieczenstwo` | przed publikacją | audyt sekretów, API i formularza |
| `analityka` | M7 | pomiar: Vercel Analytics, GA4 przez GTM, Search Console, sitemap, pixel |
| `sanity` | M8 | panel edycji treści |
| `/strona` | po instalacji | status, wznowienie, rozwój |

## Sprawdź instalację

```bash
find .claude -maxdepth 3 -type f | sort
```

W Claude Code:

```plain text
Sprawdz, czy paczka kursowa jest zainstalowana poprawnie: skille, komenda /strona i materialy kursowe. Jesli czegos brakuje, napraw instalacje zgodnie z README paczki.
```

## Jak działa skill

Skill to instrukcja, którą Claude Code wczytuje wtedy, gdy jest potrzebna. Dzięki temu nie musisz za każdym razem tłumaczyć mu procesu od zera.

Możesz uruchamiać skille wprost:

```plain text
Uruchom skill karty.
```

Albo opisać cel:

```plain text
Chce przygotowac trzy karty strony przed budowa.
```

Claude Code powinien rozpoznać, że pasuje skill `karty`.

## Efekt końcowy M2

Wiesz, że kurs nie jest zbiorem przypadkowych promptów. To system: kontekst -> karty -> design -> obrazy -> budowa -> publikacja -> analityka -> treść (CMS) -> rozwój.
