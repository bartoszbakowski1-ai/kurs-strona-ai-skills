# M2 - Framework: jak dzialaja Twoje skille

## Cel

Rozumiesz, co masz w paczce kursowej i ktory skill do czego sluzy.

## Mapa skilli

| Skill | Kiedy uzyc | Efekt |
|---|---|---|
| `kontekst` | M1 | folder `kontekst/` |
| `karty` | M3 | folder `karty/` z trzema kartami |
| `design` | M4 | system wygladu i anti-ai-look |
| `obrazy` | M4 | realne zdjecia i assety |
| `zbuduj-strone` | M5 | strona-system: strona glowna + podstrony + formularz |
| `sprawdz-kod` | M5/M6 i po zmianach | build, mobile, jakosc, bezpieczenstwo |
| `bezpieczenstwo` | przed publikacja | audyt sekretow, API i formularza |
| `sanity` | M7 | panel edycji tresci |
| `/strona` | po instalacji | status, wznowienie, rozwoj |

## Sprawdz instalacje

```bash
find .claude -maxdepth 3 -type f | sort
```

W Claude Code:

```plain text
Sprawdz, czy paczka kursowa jest zainstalowana poprawnie: skille, komenda /strona i materialy kursowe. Jesli czegos brakuje, napraw instalacje zgodnie z README paczki.
```

## Jak dziala skill

Skill to instrukcja, ktora Claude Code wczytuje wtedy, gdy jest potrzebna. Dzieki temu nie musisz za kazdym razem tlumaczyc mu procesu od zera.

Mozesz uruchamiac skille wprost:

```plain text
Uruchom skill karty.
```

Albo opisac cel:

```plain text
Chce przygotowac trzy karty strony przed budowa.
```

Claude Code powinien rozpoznac, ze pasuje skill `karty`.

## Efekt koncowy M2

Wiesz, ze kurs nie jest zbiorem przypadkowych promptow. To system: kontekst -> karty -> design -> obrazy -> budowa -> kontrola -> publikacja -> rozwoj.
